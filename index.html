アルゴリズム可視化ツール

各種アルゴリズムの動作をステップごとに可視化し、背景知識も確認できます。

Edit Distance の背景知識

レーベンシュタイン距離（編集距離）は、2つの文字列間で1文字の挿入・削除・置換を操作して一致させる際の最小操作回数を表します。動的計画法を利用し、2次元のDPテーブルを埋めることで計算します。

ステップの流れ：
1. 文字列 s1 の長さを N、s2 の長さを M として、(N+1)×(M+1) のテーブルを用意。
2. dp[0][j] = j （空文字→s2 の接頭辞までの挿入回数）、dp[i][0] = i（s1 の接頭辞→空文字までの削除回数）を初期化。
3. 各セル dp[i][j] を「削除」「挿入」「置換」の3通りのケースから最小値を選択して埋める。
4. 最終的に dp[N][M] が編集距離となる。

応用例：
- スペルチェックや綴り修正
- DNA 配列の類似度計算

0/1 Knapsack の背景知識

0/1ナップサック問題は、限られた容量の中で価値が最大となるように品物を選択する最適化問題です。動的計画法で解く場合、品物数を N、容量を W として (N+1)×(W+1) の DP テーブルを構築します。

ステップの流れ：
1. dp[i][j] を「品物 i までを検討したとき、容量 j で得られる最大価値」と定義。
2. 初期状態 dp[0][*] = 0（品物 0 個なら価値 0）。
3. 品物 i を「使わない」場合 dp[i-1][j]、「使う」場合 dp[i-1][j - w[i]] + v[i] のうち大きい方を選択。
4. 最終的に dp[N][W] が最適解。

応用例：
- リソース割り当て、投資配分
- メモリ管理、ナップサック暗号

Sorting の背景知識

ソートは、データを一定の順序（昇順・降順など）で並べ替える基本操作です。代表的なアルゴリズムには、バブルソート、挿入ソート、選択ソート、クイックソート、マージソートなどがあります。

各アルゴリズムの特徴：
- バブルソート：隣接要素を比較して交換しながら大きい値を末尾に「バブル（泡）のように浮かばせる」。最悪計算量 O(N²)。
- 挿入ソート：未整列部分から1つずつ取り出し、整列済み部分に適切な位置に挿入していく。ほぼ整列済みの場合に高速。O(N²)。
- 選択ソート：未整列部分から最小値を選択し、先頭の要素と交換して整列部分を増やす。O(N²)。
- クイックソート：分割統治法に基づき、ピボットを選んで配列を2つの部分に分割し、それぞれを再帰的にソート。平均 O(N log N)。
- マージソート：分割統治法で配列を半分に分割し、再帰的にソート後にマージ。安定かつ O(N log N)。

Searching の背景知識

探索は、配列やリストの中から特定のキーを見つける操作です。代表的な手法には線形探索、2分探索、番兵法があります。

各アルゴリズムの特徴：
- 線形探索：先頭から順番にキーを比較していく単純探索。ソート不要だが、最悪探索量 O(N)。
- 2分探索：ソート済み配列を半分に分割し、中央要素と比較して探索範囲を半減させる。O(log N)。
- 番兵法：配列の末尾に探索キーを「番兵」として一時的に追加し、比較回数を最適化した線形探索。境界チェック不要。

Forward アルゴリズム (HMM) の背景知識

隠れマルコフモデル（HMM）は、隠れた状態列と観測列の確率モデルです。Forward アルゴリズムは、観測列が与えられたときの尤度（全ての隠れ状態列を総和した確率）を計算する動的計画法です。

ステップの流れ：
0. 状態遷移行列 A、観測確率行列 B、初期状態分布 π、観測列 O が与えられる。
1. α[0][i] = π[i] * b_i(O[0]) で初期化。
2. 逐次 t=1..T-1 について、α[t][j] = (Σ_i α[t-1][i] * a_{ij}) * b_j(O[t]) を計算。
3. 最終的に Σ_i α[T-1][i] が観測列 O の確率（尤度）となる。

応用例：
- 音声認識、自然言語処理
- 生体信号解析、時系列データのモデル化

Viterbi アルゴリズム (HMM) の背景知識

Viterbi アルゴリズムは、隠れマルコフモデルにおいて、与えられた観測列の下で最尤の隠れ状態列（最適パス）を動的計画法で求める手法です。

ステップの流れ：
0. δ[0][i] = π[i] * b_i(O[0])、ψ[0][i] = 0 で初期化。
1. 逐次 t=1..T-1 について、δ[t][j] = max_i { δ[t-1][i] * a_{ij} } * b_j(O[t])、ψ[t][j] = argmax_i { δ[t-1][i] * a_{ij} } を計算。
2. 最終時刻で最大の δ[T-1][i] を持つ状態が最終状態となり、ψ を逆にたどって最適パスを復元。

応用例：
- 品詞タグ付け、音声認識
- 連鎖構造の解析

線形計画法 (Simplex) の背景知識

線形計画法は、線形目的関数を線形制約の下で最大化・最小化する最適化手法です。Simplex アルゴリズムは、基本可行解から順次列を移動しながら最適解を探索します。

ステップの流れ：
1. 問題を標準形に変形し、スラック変数を導入して初期単純形表を構築。
2. ピボット操作を繰り返し、目的関数の係数が負でなくなるまで表を更新。
3. 最終的に目的関数の値が最大（または最小）となる基本可行解を得る。

応用例：
- 資源配分、スケジューリング
- ネットワークフロー最適化

ハッシュ法 (Open Addressing) の背景知識

オープンアドレス法は、衝突が発生した際にテーブル内を再探索（プロービング）して空きスロットを探す手法です。代表的な再探索法にリニアプロービング、二次探査、二重ハッシュなどがあります。

リニアプロービングの流れ：
1. キー k のハッシュ値 h = k mod m を計算。
2. スロット h から順にインクリメントして空きスロットを探す: (h + i) mod m をチェック。
3. 空きスロットが見つかるまで i を増やし続ける。
4. テーブルが満杯の場合は挿入失敗となる。

応用例：
- データベース、キャッシュ実装
- メモリ管理

スタックの背景知識

スタックは、最後に入れた要素を最初に取り出す（LIFO: Last-In, First-Out）のデータ構造です。関数呼び出しの管理や深さ優先探索などに利用されます。

主な操作：
- push(x): スタックのトップに要素 x を追加。
- pop(): スタックのトップから要素を取り出して削除。
- peek(): スタックのトップ要素を参照するが削除はしない。

応用例：
- 関数呼び出しの再帰管理
- ブラウザの戻る履歴

キューの背景知識

キューは、最初に入れた要素を最初に取り出す（FIFO: First-In, First-Out）のデータ構造です。タスクスケジューラや幅優先探索などで利用されます。

主な操作：
- enqueue(x): キューの末尾に要素 x を追加。
- dequeue(): キューの先頭から要素を取り出して削除。
- peek(): キューの先頭要素を参照するが削除はしない。

応用例：
- プロセススケジューリング
- 印刷ジョブ管理

ハッシュ法 (チェイン法) の背景知識

チェイン法は、ハッシュ衝突が起きたときに各スロットにリスト（チェイン）を作成し、衝突したキーをリストに追加する手法です。探索時にはチェイン内を線形走査します。

流れ：
1. キー k のハッシュ値 h = k mod m を計算。
2. スロット h のチェイン（リスト）にキー k を追加。
3. 探索時はスロット h のチェインを線形探索してキーを見つける。

応用例：
- データベース、キャッシュ実装
- 連想配列 (Dictionary)
